# Diseño del paquete documental

## Objetivo

Completar la estructura sugerida por `Readme_Docs_Sugerencia_Enlaces_Presentaciones_Demo.xlsx` y entregar documentación verificable para evaluación, operación, defensa de producto y defensa técnica.

## Estructura

- `RECURSOS/`: presentación editable, PDF y portada.
- `README.md`: ficha ejecutiva, problema, datos, variables, modelo, resultados, impacto, producción y enlaces.
- `docs/`: fuentes editables en Markdown sobre problema, metodología, datos, diccionario, arquitectura, conclusiones, validación, informe técnico, manual y guías especializadas.
- `documentation/`: PDF finales y diagrama de arquitectura en PNG.

## Documentos especializados

La guía de producto y operaciones para Esperanza se diseña como material de estudio: narrativa, responsabilidades, preguntas frecuentes, preguntas difíciles, respuestas modelo, métricas de producto, adopción, riesgos operativos y ejercicios de práctica.

La guía de infraestructura explica el monorepo, la división por servicios, Bun y Elysia, controles de seguridad, procesamiento de consultas grandes, persistencia y caché, integración de Python para aprendizaje automático, comparación con un backend completamente Python y una ruta de escalamiento por etapas.

## Principios

- No afirmar que una tecnología garantiza seguridad o rendimiento por sí sola.
- Diferenciar riesgo indicativo de prueba de corrupción.
- Vincular afirmaciones técnicas con código, pruebas o configuración existentes.
- Distinguir capacidades actuales de recomendaciones futuras.
- Conservar fuentes editables para actualizar los PDF.

## Validación

- Comprobar existencia y nombres exactos.
- Validar estructura de cada PDF y número de páginas.
- Revisar visualmente portada, páginas intermedias y tablas.
- Verificar enlaces y consistencia de cifras.
- Mantener `RECURSOS/` con únicamente los tres archivos solicitados.
